# AI-Assisted Maintenance Readiness

## One-Line Offer

Raid Guild helps mature software teams turn hard-to-maintain legacy systems into agent-ready, human-governed maintenance environments with cleaner code, clearer docs, operational runbooks, and a deployed AI-assisted workflow stack.

## Core Buyer

This offer is strongest for teams with live software, real users, and too much operational knowledge trapped in old repos, stale docs, Discord history, and a few contributor brains.

Primary target buyers:

- **Mature web3 protocols** with active infrastructure but a smaller maintainer base than they once had.
- **DAOs preparing for stewardship transition** to a new maintainer, working group, vendor, or foundation structure.
- **Small product teams with large maintenance surfaces** where recurring support, bug triage, and dependency work crowd out product development.
- **Ecosystem funders** who want public-goods infrastructure to remain usable without permanently funding a large team.

## Pain And Opportunity

These buyers often feel the same cluster of problems:

- No one has a complete map of active repos, apps, docs, infra, access, billing, and support paths.
- Dependency updates feel risky, so they get deferred until the system is more fragile.
- New maintainers take too long to onboard because the real operating model is undocumented.
- Support requests arrive through scattered channels and lack a clear intake path.
- AI coding tools look useful, but the repo and process are not structured enough to use them safely.
- Production responsibility is too concentrated in a few historical contributors.

The opportunity is to give the client a maintainable operating surface: smaller, documented, agent-ready, and governed by clear human approval gates.

## Offer Narratives

**"Your protocol is live. Your maintenance model should be too."**  
For mature protocols that still matter but no longer have a large core team. The offer is about continuity, not novelty.

**"AI-assisted maintenance without handing production to an agent."**  
For cautious technical buyers. The agent handles triage, investigation, branch preparation, reporting, and context retrieval; humans approve decisions.

**"Turn legacy complexity into an agent-ready operating surface."**  
For teams with sprawling repos, docs, Discords, and infrastructure. The work starts with simplification before automation.

**"One maintainer, more leverage."**  
For lean teams. The package helps one maintainer spend less time rediscovering context and more time making good decisions.

**"A transition packet plus the machine to keep it alive."**  
For stewardship handoffs. The client receives both durable documentation and a live workflow system that preserves future operational memory.

## What Is The Refactory?

The Refactory is a deployable AI-assisted maintenance control plane developed from the Superprism and Dark Factory work and hardened during the DAOhaus raid.

It is not a fully autonomous maintainer. It coordinates human maintainers, coding agents, project knowledge, GitHub, Discord, and deployment context through a structured workflow.

A typical Refactory instance includes:

- Site/API control plane for requests, tasks, workflows, artifacts, skills, and agent sessions.
- Codex Runtime for bounded agent execution.
- Prism Memory for operational history, transcripts, digests, runtime outputs, and artifacts.
- Prism Knowledge for curated docs, SOPs, architecture notes, and stable maintainer guidance.
- Discord adapter for intake and sync.
- Recurring memory, knowledge, and Discord sync jobs.
- GitHub as the source of truth for branches, pull requests, review, and merge state.
- Railway deployment context for service health and verification.

Each client gets its own instance with its own secrets, volumes, target repos, auth, Discord configuration, and cron schedules.

## What The Package Includes

### 1. Maintenance Audit

- Inventory active repos, apps, docs, infrastructure, DNS, hosting, Discord, and automation.
- Identify active, deprecated, and historical systems.
- Map access, billing, ownership, and single points of failure.
- Separate routine maintainer work from exceptional changes.

### 2. Codebase And Repo Readiness

- Simplify repo structure where needed.
- Reduce package, alias, and dependency indirection.
- Colocate feature-specific code.
- Mark legacy repos and packages clearly.
- Modernize high-risk dependencies when scoped.

This is often the AI readiness sprint: agents work better when the codebase has a clear shape.

### 3. Agent-Ready Documentation

- Root documentation map.
- Architecture overview.
- Environment and deployment docs.
- Symptom-based debugging guide.
- Domain glossary and verification guide.
- Local `AGENT.md` files near key code areas.
- Copyable AI maintainer prompt for future agent sessions.

### 4. Refactory Deployment

- Deploy a client-specific Refactory instance.
- Configure target repos, environments, intake, memory, and knowledge.
- Configure recurring sync and indexing jobs.
- Complete required Codex Runtime auth and smoke tests.
- Establish issue lifecycle and human approval gates.

### 5. SOP And Transition Packet

- Operating model.
- System architecture.
- Repository and application inventory.
- Infrastructure, access, and dependency matrix.
- Operational runbooks.
- AI-assisted maintenance workflow guide.
- Known risks, gaps, and recommended next actions.
- Stewardship handoff checklist.

### 6. Maintainer Enablement

- Walkthrough of the Refactory instance.
- Training on intake, triage, branch review, and escalation.
- Guidance on what agents can and cannot do.
- Optional first-month support while the workflow becomes normal.

## What Buyers Get

- A smaller and clearer maintenance surface.
- A documented system new maintainers can understand.
- A deployed AI-assisted workflow stack.
- A human-in-the-loop issue lifecycle.
- Faster triage and investigation paths.
- Durable operational memory and curated knowledge.
- Explicit access and escalation boundaries.
- A transition-ready maintenance packet.
- A foundation for ongoing support or maintainer retainers.

## What This Is Not

- Not fully autonomous production maintenance.
- Not a replacement for human technical ownership.
- Not a security audit, unless separately scoped.
- Not a contract upgrade engagement, unless separately scoped.
- Not a promise that legacy systems need no future investment.

The offer is strongest when framed as leverage, continuity, and risk reduction.

## Package Shapes To Test

- **Diagnostic Sprint:** inventory, gap report, readiness score, and implementation roadmap.
- **Readiness Sprint:** surface cleanup, agent-ready docs, SOP packet, access matrix, and maintainer prompt.
- **Refactory Implementation:** deployed instance, memory/knowledge setup, intake workflow, human gates, and operator handoff.
- **Ongoing Maintenance Partnership:** monthly maintainer retainer, dependency review cadence, support triage, release support, knowledge promotion, and quarterly risk/access review.
